# [Walkthrough] ヒーロー遷移の解除とフェードへの復帰

複雑な要素移動を伴う「ヒーロー遷移」に代わり、ユーザーの要望に基づき標準的なフェードアニメーションを再実装しました。

## 変更内容

### ヒーロー要素のリンク解除
- `src/pages/index.astro`, `src/pages/live.astro`, `src/pages/apps/web/index.astro` の各ファイルから、アイコン・タイトル・コンテナに付与されていた `transition:name` 属性をすべて削除しました。

### 遷移スタイルのクリーンアップ
- `src/layouts/BaseLayout.astro` から、独自のアニメーション速度、イージング、および `z-index` 制御を行っていた `::view-transition-*` 関連のグローバル CSS を削除しました。

## 検証結果

### 動作確認
- ホーム ↔ 各ページ間の遷移において、要素の物理的な移動（飛行）が発生せず、ページ全体がスムーズにクロスフェードすることを確認しました。
- Astro のデフォルト機能である `transition:animate="fade"` が正しく機能し、視覚的なガタつきが解消されていることを確認しました。
