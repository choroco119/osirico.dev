# ヘッダーレイアウトの修正とモバイル対応の強化

ソーシャルリンクの追加に伴うヘッダー構造の変化により、モバイル表示での要素の重なりやデスクトップでの不自然な余白が発生しています。本計画では、ヘッダーを堅牢な構造に再設計し、レスポンシブ対応を全ページで一貫させるとともに、YouTubeチャンネルへのリンクを追加します。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> - **モバイルナビゲーション**: 768px未満の画面では、要素の重なりを防ぐため、ナビゲーションメニューのフォントサイズ調整や、アイコンの配置を最適化します。
> - **レイアウトの安定化**: すべてのページで共通のヘッダーCSSが適用されるよう、`BaseLayout.astro` にメディアクエリを集約して管理します。

## 提案される変更点

### コアレイアウトの改善

#### [MODIFY] [BaseLayout.astro](file:///c:/Users/kohei/.gemini/antigravity/scratch/osiricoop-dev/src/layouts/BaseLayout.astro)
-   **構造上の変更**: 
    -   ロゴと右側のグループ（ナビゲーション＋ソーシャル）を分離し、`space-between` を維持しつつ、各グループ内の要素間の距離を最適化します。
    -   YouTubeアイコンを追加し、ホバー時にブランドカラーの赤色で発光するエフェクトを設定します。
-   **CSSの強化**:
    -   総合的なメディアクエリ (`@media (max-width: 768px)`) を実装。
    -   モバイル時にメニューが2行に分かれないよう、または分かれても美しく見えるよう調整します。

-   **CSS**: ページ固有で定義されていた不要なメディアクエリ（ヘッダーのアライメントを個別に修正しようとしていたもの）を削除し、`BaseLayout.astro` のグローバルな定義に従うようにします。

## 検証計画

### 自動テスト (Browser Subagent)
-   デスクトップ (2560px, 1920px) での配置確認。
    - ロゴが左端、ナビとソーシャルが右側にまとまっているか。
-   モバイル (375px) での重なり確認。
    - 要素が重ならず、タップしやすい間隔が保たれているか。

### 手動確認
-   ユーザーに実際のモバイル端末（iPhone等）で表示崩れが解消されたかを確認していただきます。
