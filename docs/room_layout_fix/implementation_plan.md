# オブジェクト複数選択機能の実装計画

オブジェクトの複数選択（Shift+クリックおよび範囲ドラッグ）を実装し、操作性を向上させます。また、頂点とオブジェクトが混在して選択されないように制御します。

## ユーザーレビューが必要な事項

- **範囲ドラッグ中の挙動**: 範囲ドラッグ開始時に既存の選択をクリアするか、あるいはShiftを押しながらのドラッグで「追加選択」にするか。今回は一般的なエクスプローラー形式（通常ドラッグでクリア＆新規、Shift+ドラッグで追加）を想定します。
- **複数個移動のコリジョン**: 複数選択された状態でドラッグ移動する際、1つでも衝突が発生した場合に全員の移動をキャンセルするか、移動可能なものだけ動かすか。今回は「一貫性を保つため全員キャンセル」または「衝突した個別オブジェクトのみ赤枠表示（移動は許容）」のいずれかが考えられますが、既存の「cancelMove」ロジックに合わせて「全員キャンセル」を基本とします。

## 変更内容

### `public/apps/web/room-layout/script.js` の修正

#### 1. 状態管理の拡張
- `STATE.interaction.marqueeStart` を活用し、ドラッグ開始地点を記録します。

#### 2. `handleMouseDown` の修正
-  panning（右クリック/Alt）や頂点/オブジェクト単体クリックでない場合、範囲選択（Marquee）を開始するようにします。

#### 3. `handleMouseMove` の修正
- 範囲選択中の場合、`#selection-marquee` の位置とサイズを更新します。
- 複数選択中のドラッグ（`dragTarget` が複数）の場合、選択されている全オブジェクトの座標を更新します。

#### 4. `handleMouseUp` の修正
- 範囲選択を確定し、矩形内にあるオブジェクトを `STATE.selection.ids` に追加します。
- **制約**: オブジェクトが1つでも選択された場合、頂点の選択（`type === 'vertex'`）を解除します。

#### 5. `selectObject` の修正
- `event.shiftKey` を受け取れるようにし、複数選択のトグル処理（既にあれば削除、なければ追加）を実装します。
- 頂点選択中にオブジェクトを選択した場合は、頂点選択をクリアします。

#### 6. `renderObjects` の修正
- すべての選択済みオブジェクトを視認しやすくハイライトします（既存の `selected` クラスを利用）。

### `public/apps/web/room-layout/styles.css` の修正
- `.selection-marquee` が正しく表示されるよう、`z-index` や `display` を調整します。

## 検証計画

### 手動検証
- [ ] Shiftキーを押しながら複数の家具をクリックし、すべてが選択状態（青枠）になることを確認。
- [ ] 何もないところからドラッグし、範囲選択枠が表示されることを確認。
- [ ] 範囲選択枠で囲った複数の家具が選択状態になることを確認。
- [ ] 複数選択された家具のどれか1つをドラッグした際、全員が同期して移動することを確認。
- [ ] 家具を選択した状態で壁の頂点を選択しようとした際、家具の選択が解除されることを確認。
- [ ] 家具が選択に含まれている場合、頂点の詳細パネルが表示されないことを確認。
