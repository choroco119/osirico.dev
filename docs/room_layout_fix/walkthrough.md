# 家具追加挙動と複数選択機能の改善

家具を追加した際のマウス追従を廃止し、操作性を向上させるための複数選択機能を実装しました。また、キャンバスの外観をクリーンに整理しました。

## 変更内容

### `script.js` / `styles.css` / `room-layout.astro` の修正
- **オブジェクトの複数選択機能**:
    - **Shift + クリック**: オブジェクトを選択範囲に追加/解除できます。
    - **範囲ドラッグ (Marquee)**: 何もない場所からドラッグすることで、矩形範囲内のオブジェクトを一括選択できます。
    - **複数同時移動**: 複数選択された状態でドラッグ、または矢印キー操作（Shift併用で100mm単位）を行うと、全オブジェクトが同期して移動します。
    - **選択の排他制御**: オブジェクトを選択した場合、壁の頂点（ドット）の選択は自動的に解除され、オブジェクトと頂点が同時に選択されることはありません。
- **家具追加挙動**: `addObject` 関数で新しい家具を追加する際、マウスへの自動追従を停止し、中央に即時配置・確定するようにしました。
- **名称ルール**: 形状ごとに固有の名称（矩形: R1, R2...、円形: C1, C2...）を自動生成するようにしました。
- **外観の整理**:
    - キャンバスを囲んでいた青い枠、濃い背景色、シャドウ、および原点マーカーを削除しました。
    - サイドバーの「表示設定」パネルを削除しました。
- **不具合修正**:
    - 円形（circle）が追加時に描画されない、または極小で見えない問題を改善しました。
    - 家具同士を隙間なく並べられるよう、衝突判定に `0.01mm` の許容誤差を導入しました。

## 検証結果

ブラウザ上の動作検証（サブエージェントによる自動検証含む）により、以下の動作を確認しました：
1. **複数選択**: Shiftキーを使用、またはマウスドラッグによる範囲指定で、複数の家具が期待通りに同時選択（ハイライト）される。
2. **連動移動**: 複数選択された家具の1つを動かすと、他の選択中の家具も同じ距離だけ移動する。衝突判定も維持される（1つでも衝突すれば移動がキャンセルされる）。
3. **排他選択**: 家具を選択した状態で壁の頂点をクリックすると、家具の選択が解除される。
4. **シンプルUI**: 不要な青枠や設定パネルが消え、広々としたキャンバスで作業できる。

---
以上の修正により、より直感的な家具のレイアウトと、効率的な複数オブジェクトの操作が可能になりました。
