# Ctrl+ドラッグ複製時の挙動改善

Ctrl+ドラッグでオブジェクトを複製する際、以下の挙動を実現するように修正します。
1.  **移動対象の切り替え**: 複製を実行した瞬間、ドラッグの制御対象を「元のオブジェクト」から「新しく作成されたオブジェクト」に切り替えます。これにより、マウスの下にあるオブジェクトがそのまま移動しているように見えます。
2.  **一時的な衝突判定の無効化**: 複製されたオブジェクトは、最初にマウスを離す（ドラッグを終了する）まで、一時的に衝突判定（他のオブジェクトや壁との接触制限）を無効化します。これにより、重なった状態からの引き出しがスムーズになります。

## 変更内容

### [script.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/osiricoop-dev/public/apps/web/room-layout/script.js)

#### [MODIFY] `duplicateSelected()`
- 複製されたオブジェクトに一時的なフラグ `tempDisableCollision = true` を付与します。
- 複製後のオブジェクトの対応マップ（元ID -> 新オブジェクト）を返すようにします。

#### [MODIFY] `handleMouseMove()`
- `duplicateSelected()` が実行された直後に、現在ドラッグ中の対象（`STATE.interaction.dragTarget.target`）を、対応する新しく生成されたオブジェクトに差し替えます。
- これにより、その後の移動処理が「新規オブジェクト」に対して行われるようになります。

#### [MODIFY] `checkCollisions()` & `testCollisionWithAny()`
- `tempDisableCollision` が `true` のオブジェクトについては、衝突計算をスキップするように条件を追加します。

#### [MODIFY] `handleMouseUp()`
- ドラッグ終了時に、すべてのオブジェクトから `tempDisableCollision` フラグを削除します。

## 検証プラン

### 手動検証
1.  オブジェクトを配置し、Ctrlキーを押しながらドラッグを開始する。
2.  複製された方のオブジェクトがマウスに追従して移動することを確認。
3.  移動中、他のオブジェクトや壁を突き抜けて自由に移動できる（衝突判定がOFFになっている）ことを確認。
4.  マウスを離した後、再度そのオブジェクトをドラッグした際には、通常の衝突判定が有効になっていることを確認。
5.  複数選択状態での Ctrl+ドラッグ でも、選択されていた全オブジェクトの複製がマウスに追従し、すべて衝突判定が一時的にOFFになることを確認。
